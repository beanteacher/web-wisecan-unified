## 사용자 동선 (User Flow)

### 일반 사용자 동선
```
진입 (메인) → 탐색/검색 → 목록 → 상세
  → [프로젝트 핵심 기능] → 전환(구매/신청 등) → 완료
```

### 관리자 동선
```
관리자 로그인 → 관리자 대시보드 → 콘텐츠 등록/관리 → 운영 관리
```

### 비회원 동선
```
메인 → 콘텐츠 탐색 → 상세 → 로그인/회원가입 유도 → [일부 기능은 로그인 필요]
```

## 핵심 기능 인터페이스 설계 지침

### 프로젝트 핵심 기능 A UI (프로젝트별 정의)
- 핵심 기능 진입점(CTA 버튼)을 상세 페이지에 배치
- 모달 또는 전체화면 전환 고려
- 로딩 상태 표시 필수

### 프로젝트 핵심 기능 B UI (프로젝트별 정의)
- 핵심 기능 B 진입점(CTA 버튼)을 적절한 페이지에 배치
- 비로그인 시 로그인 유도 모달 (로그인 필요 기능인 경우)
- 단계별 플로우가 있는 경우 온보딩 플로우 설계
- 결과 화면 레이아웃 및 상태 표시 설계

## Figma 컴포넌트 라이브러리 구성

```
Components/
├── Foundation
│   ├── Colors        ← 컬러 스와치
│   ├── Typography    ← 폰트 스타일
│   └── Spacing       ← 간격 기준
├── Elements
│   ├── Button        ← Primary, Secondary, Ghost
│   ├── Input         ← Text, Select, Checkbox, Radio
│   ├── Badge / Tag   ← 카테고리, 할인율 등
│   └── Icon          ← 아이콘 세트
├── Components
│   ├── Navbar        ← PC / Mobile 네비게이션
│   ├── ProductCard   ← 상품 카드 (썸네일, 가격, 브랜드)
│   ├── Modal         ← 기본 모달, 3D 뷰어 모달
│   ├── Carousel      ← 배너, 상품 슬라이드
│   └── Pagination    ← 페이지 네비게이션
└── Patterns
    ├── ProductGrid   ← 상품 목록 그리드
    ├── FilterPanel   ← 필터 사이드바
    └── CheckoutFlow  ← 결제 플로우
```

## 반응형 브레이크포인트

| 구분 | 범위 | 레이아웃 |
|------|------|----------|
| Mobile | 375px ~ 767px | 1컬럼, 하단 탭바, 햄버거 메뉴 |
| Tablet | 768px ~ 1279px | 2컬럼 조정형, 접이식 사이드바 또는 상단 내비게이션, 그리드 2~3열 |
| Desktop | 1280px ~ | 사이드바 + 컨텐츠 영역, 그리드 3~4열 이상 |

### 디바이스별 설계 필수 사항

- 각 페이지는 **Mobile / Tablet / Desktop 3개 프레임**을 모두 MCP 로 생성한다 (Tablet 생략 금지).
- Tablet 은 단순히 Mobile 을 늘리거나 Desktop 을 줄인 결과가 아니다 — 컬럼 수·내비게이션·여백·타이포 스케일을 중간값으로 재조정한다.
- 하단 탭바는 Mobile 전용. Tablet/Desktop 은 상단 내비게이션 또는 사이드바를 사용한다.
- 그리드: Mobile 1열 → Tablet 2~3열 → Desktop 3~4열 이상 단계적 확장.
