## 1. 한 줄 목표 / 기간 / 인력 가정

- **목표** — 2026-06-01 ~ 2026-10-30 (**22주 / 11 스프린트 / 약 5개월**) 사이에 MVP를 출시한다. KPI는 `01_PRD §성공 지표 K1·K2·K3`.
- **인력 가정** — PM 1, BE 3, FE 2, 디자이너 1, QA/운영자 1 (= 8명, 코어 개발 5명).
- **처리 용량** — 코어 5명 × 5 man-day/주 × 22주 = **550 man-day**. 기능 정의서 추정치 300~480 man-day 대비 14~45% 버퍼.
- **예산 가정** — 인건비 외 외부 비용: PG(자동충전·후불 계약), 카카오/RCS 중계사 계약비, IDC 서버 임대(자체 서버 + 단일 IDC 이중화). 세부 단가는 자원/예산 트래커(§9)에서 갱신.

## 8. 게이트 (릴리즈 통과 기준)

각 마일스톤 종료 + 베타→GA 전환 시점에 4종 게이트를 통과해야 한다.

### 8.1. 기능 게이트
- 해당 마일스톤의 모든 WBS DoD 충족
- 핵심 액션의 E2E 테스트 그린 (가입·발송·결제·이력 5종)
- ⏸ 보류 항목(구독·세금계산서·특수부가·사설 연동)은 비활성 처리 확인

### 8.2. 품질 게이트
- 단위 테스트 커버리지 라인 ≥ 70%, 핵심 도메인 ≥ 85% (`.claude/rules/testing.md`)
- 회귀 테스트 0 fail
- 정적 분석 critical 0건

### 8.3. 운영 게이트 (NFR 인용)
| 항목 | 목표 | 출처 | 측정 메트릭 (정본 `06 §2·§3`) |
|---|---|---|---|
| 발송 API 가용성 | ≥ 99.5% | NFR-AVAIL-001 | `send_api_5xx_total` / `send_api_total` → 알림 `SendApiAvailabilityBelow995` |
| 발송 API P95 응답 | ≤ 1,000ms | NFR-PERF-002 | `send_api_duration_seconds` P95 → 알림 `SendApiP95Over1s` |
| 발송 테이블 적재 성공률 | ≥ 99.9% | NFR-RELY-001 | `send_table_insert_total{result}` → 알림 `SendTableInsertSuccessBelow999` (P0) |
| 자동충전 결제 실패율 | ≤ 1% | NFR-RELY-203 | `autocharge_attempt_total{result}` → 알림 `AutochargeFailureRateOver1Percent` |
| 1:1 문의 처리 시간 | ≤ 24h | PRD §K3 보조 | `cs_ticket_open_age_hours` → 알림 `CsTicket24hBreach` |
| 사업자 전환 심사 평균 | ≤ 24h | PRD §K1 보조 | `admin_review_age_seconds{type="business"}` → 알림 `BusinessReviewSla24hBreach` |

### 8.4. 보안 게이트
- TLS 1.2+ 강제, HSTS, mTLS 내부
- 운영자 2차 인증 + 신뢰 IP 강제
- API Key OS 키체인 저장 확인
- MCP 비노출 화이트리스트 단위 테스트 통과
- 감사 로그 append-only + 5년 보존 확인
- OWASP Top 10 셀프 점검 (SQLi·XSS·CSRF·파일 업로드·Rate Limit 5종) 통과

## 11. PM 운영 원칙 인용 (`pm/CLAUDE.md` 발췌)

- **계획 우선** — 구현 지시 전에 계획 문서 먼저. 계획 없는 실행 금지.
- **가치 중심** — Task가 아닌 Outcome 기준 백로그.
- **측정 가능** — 모든 DoD는 수치/검증 가능 형태.
- **증거 중심** — 완료 보고에 실제 산출물 경로 필수. placeholder(TBD·추후 작성) 금지.
- **변경 통제** — 스프린트 중 무분별한 스코프 추가 금지. 긴급 변경은 근거/영향/대응안 함께 기록.
- **페르소나 참조** — Sub Agent 지시 전 각 `AGENTS.md` 검토.
