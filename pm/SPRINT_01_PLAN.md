# SPRINT 1 계획서

## Sprint Goal
인증 시스템과 API Key 관리 기반을 구축하고, 랜딩 페이지와 핵심 레이아웃을 완성한다.

## 기간
Sprint 1 (2주)

## 포함 범위
- 디자인 시스템 초기화 + 랜딩/인증/대시보드 와이어프레임
- 회원가입/로그인 API + JWT 인증
- API Key CRUD API
- 랜딩 페이지 구현
- 공통 레이아웃 (Header/Footer/Sidebar)

## 제외 범위
- 도구 프록시 API (Sprint 2~3)
- 도구 UI 화면 (Sprint 3~4)
- 사용량 상세 통계 (Sprint 5+)
- 소셜 로그인, 요금제

## 리스크

| 리스크 | 영향 | 완화책 |
|--------|------|--------|
| UI/UX handoff 지연 → FE 착수 지연 | 일정 밀림 | 디자인 시스템 1일차 완료 → 레이아웃 FE 선착수 |
| JWT 구현 복잡도 | BE 일정 지연 | jjwt 라이브러리 기본 설정으로 시작, refresh 토큰은 최소 구현 |
| API Key 보안 설계 | 보안 이슈 | Key는 발급 시 1회만 노출, DB에는 SHA-256 해시만 저장 |
