# PRD — WiseCan 통합 메시징 서비스

> 작성일: 2026-04-30
> 상태: 1차 초안
> 참조: `pm/design/SRS.md` (요구사항 명세서)

---

## 1. 개요 / 비전

**한 줄 요약** — "AI 시대에 가장 저렴하고 가장 통합 친화적인 한국 메시지 발송 서비스"를 만든다.

**비전**
- 회원은 SMS·LMS·MMS·카카오 알림톡·RCS를 **하나의 계정·하나의 키·하나의 잔액**으로 발송한다.
- 통합 진입점을 SDK·**MCP**·**CLI** 3종으로 동시 제공해, AI Agent 자동화 워크플로우의 **토큰 비용을 5~10배 절감**한다.
- 회원은 중계사·라우팅 같은 내부 인프라를 알 필요 없이 채널만 선택한다 (`SMS / LMS / MMS / 카카오 알림톡 / RCS`).

## 2. 왜 만드는가 (Why)

### 2.1. 시장 문제

| 문제 | 현황 | 본 서비스의 답 |
|---|---|---|
| **AI Agent 통합 시 토큰 비용 폭증** | 기존 메시지 SaaS는 MCP만 제공하거나 CLI 자체가 없음. AI Agent가 발송·이력 조회를 호출할 때마다 4,000~6,000 토큰 소모. | MCP + CLI를 1:1 동등하게 제공. 자동화 루프는 CLI(`wsc ...`)로 위임 → **600~900 토큰**으로 5~10배 절감. |
| **카카오/RCS 중계사 다단계 라우팅이 회원에게 노출됨** | LG CNS·KT·인포뱅크 라우팅 식별자가 응답·이력에 새어 나가 회원이 혼란 + 운영 협상력 약화. | 중계사·라우팅·내부 채널 ID를 **응답·UI·로그·이력 어디에도 노출하지 않음** (`RQ-SEND-308`). 회원 인지 단위는 채널까지. |
| **잔액 부족 시 발송 전체 차단** | 잔액 소진 → 모든 발송 일괄 차단으로 운영 클레임·재시도 비용 발생. | "사전 평가 + 분기"로 전환. 자동결제→후불→부분 발송/취소 순으로 분기, **이미 적재된 발송은 완주** (`RQ-PAY-006~011`). |
| **운영 키 전환 = 운영자 전수 검토** | 신규 회원 폭증 시 운영 병목. | **조건 자동 승인 + 위험 신호 시 수동 검토** 하이브리드 (`RQ-KEY-009~011`, `RQ-ADMIN-603`). |
| **체험을 위해 가입·심사·KISA·결제까지 거쳐야 함** | 도입 결정 전에 손으로 만져볼 방법이 없어 전환율 손실. | 비회원 **체험 모드** — 더미 데이터로 회원과 동일 UI 즉시 조작, 실발송·결제만 차단 (`RQ-TRY-001~016`). |

### 2.2. 기존 SMS17 운영에서 학습한 부채

- 회사 마스터 권한 모델이 명확하지 않아 담당자 퇴사 시 권한 이관 혼선 → **A/B안 분기 명시 후 1차 결정** (`§14.A`).
- 카카오 직접 대행 vs 중계사 — 1차에는 **중계사 연동(B안)** 으로 확정해 외부 발송 시스템 책임으로 분리 (`§14.D`, `RQ-SEND` 외부 분리).
- 무통장입금·일괄 무료 발송 등 어뷰징 진입점 차단 (`RQ-PAY-007`, `RQ-SEC-001`).

## 3. 누구를 위해 (For Whom)

### 3.1. 1차 타깃 (MVP 사용자)

| 페르소나 | 핵심 니즈 | 본 서비스가 주는 가치 |
|---|---|---|
| **개인 개발자 / 1인 사업자** (`MEMBER`) | 개인 인증만으로 빠르게 SMS·카카오 발송, 코드 통합 | 본인 인증 → 발신번호 등록 → API Key 발급까지 셀프서비스. SDK·CLI로 즉시 발송. |
| **회사 마스터** (`COMPANY_MASTER`) | 사업자 등록증으로 회사 단위 발신번호·결제 통합, 직원 권한 분리 | 회사 마스터 + 하위 계정 모델, 키별 스코프(`RQ-KEY-101`)로 직원별 권한 통제. |
| **회사 하위 계정 (직원)** (`COMPANY_MEMBER`) | 회사 발신번호로 마케팅·알림 발송, 자기 사용량만 조회 | 키별 조회 범위(`scope:key` 기본)로 데이터 격리. 마스터가 부여한 권한 내 발송. |
| **AI Agent를 운영하는 개발자** | 대화형으로 발송·이력 조회를 자동화하되 토큰 비용은 줄이고 싶음 | MCP(자연어 단발) + CLI(자동화 루프) 동등 제공. 보안 민감 액션은 MCP 비노출(`RQ-MCP-014`). |
| **고려 중인 비회원** | 가입 전에 실제 화면을 만져보고 싶음 | 체험 모드 — 더미 발송·이력·결제까지 회원과 동일 UI에서 시뮬레이션. |

### 3.2. 1차 비대상

- 특수부가 사업자 (KISA 식별 코드) — 1차 범위 외 (`RQ-AUTH-201`, `RQ-CALLBACK-007`).
- 사설 연동 고객 (부동산 중개·보험사 변작 방지 증빙) — 1차 범위 외 (`RQ-AUTH-202`, `RQ-CALLBACK-203`).
- 글로벌 발송(외국 → 한국) — 데이터 모델·인증·결제 흐름은 확장 가능하게 설계만 (`RQ-ARCH-006`), MVP 미포함.

### 3.3. 운영 측 사용자

- **슈퍼 어드민** (`SUPER_ADMIN`) — 운영자 계정 발급·권한 위임·자동 승인 정책·쿨링 오프 정책 등 시스템 정책의 단일 관리 지점.
- **일반 운영자** (`ADMIN`) — 심사·CS·재무·보안 분야별 위임 권한 운영.

## 4. 무엇을 만드는가 (What — 솔루션 개요)

### 4.1. 핵심 기능 블록 (15개 분류)

```
RQ-AUTH    회원가입 / 본인인증 / 2차 인증
RQ-TRY   비회원 체험 모드 (더미 데이터 격리)
RQ-MEMBER  회사 마스터 / 하위 계정 (A안: 권한 부여형 / B안: 트리 종속형)
RQ-ADMIN   시스템 운영자 (심사·통제·재무·CS·테스트·시스템 설정)
RQ-CALLBACK  발신번호 등록·관리·KISA 사전 등록 자동 연계
RQ-KEY     API Key (테스트/운영 키, 스코프 12종, 자동 승인 + 위험 시 수동)
RQ-SDK     Python 등 라이브러리 (자기 코드 import)
RQ-MCP     AI Agent 진입점 (조회·발송, 보안 민감 액션 비노출)
RQ-CLI     셸·CI/CD·자동화 루프 진입점 (`wsc` 명령)
RQ-SEND    발송 — 정합성 검증 → 인코딩(UTF-8→EUC-KR) → 발송 테이블 적재까지
RQ-TPL     카카오·RCS 템플릿/브랜드 등록·심사·이관
RQ-PAY     결제 (선불 기본 + 자동충전·후불·구독 옵션 + 쿨링 오프 환불)
RQ-TEST    SDK/MCP/CLI 한정 테스트 키·채널별 한도 (자동 부여)
RQ-SEC     스팸 필터·이상 패턴·한도 차단
RQ-OPS     FAQ·챗봇·1:1 문의·공지
RQ-ARCH    3채널 통합 / 인코딩 / 글로벌 확장성
```

### 4.2. 책임 경계 (본 서비스 vs 외부 발송 시스템)

```
[회원/SDK/MCP/CLI 요청]
        ▼
   본 서비스 책임           외부 발송 시스템 책임
   ─────────────────       ──────────────────────
   정합성 검증              채널 라우팅 / 1:1 매핑
   인코딩 변환              폴백 실행
   (UTF-8 → EUC-KR)         실 송출 (통신사·중계사)
   발송 테이블 적재 ───────> 발송 테이블 polling
```

본 서비스는 **발송 테이블 적재까지**만 책임지고, 그 이후는 외부 발송 시스템이 수행한다 (`§7.4`, `RQ-SEND-301~308`).

### 4.3. 차별화 포인트 (기존 사이트 대비)

1. **MCP ↔ CLI 동등 (조회·발송)** — 동일 코퍼스 가이드, 동일 조회·발송 도구. AI Agent가 자유롭게 비용 최적 진입점 선택. 단, 보안·재무 민감 액션은 **의도적 MCP 비노출(RQ-MCP-014)** — AI Agent의 자연어 추론으로 인한 우발 호출(키 무단 발급, 발신번호 무단 삭제, 결제 등)을 차단. 해당 액션은 웹 콘솔 또는 CLI 명시 명령으로만 수행.
2. **API Key 스코프 12종 + 조회 범위 토글** — 발송/조회/관리 권한을 키 단위로 제어, 키 유출 시 노출 범위 자동 축소 (`§5.3`).
3. **회원 노출 추상화** — 중계사·라우팅·내부 채널 ID 0 노출 (`RQ-SEND-308`).
4. **잔액 부족 분기 처리** — 자동결제→후불→부분 발송/취소, 적재된 건은 완주 (`RQ-PAY-006~011`).
5. **운영 키 전환 자동 승인** — 운영자 검토 부담 최소화, 위험 신호만 큐로 (`RQ-KEY-009~011`).
6. **비회원 체험 모드** — 가입 결정 전 회원과 동일 UI에서 더미 시뮬레이션 (`RQ-TRY-*`).
7. **구독 쿨링 오프 환불 모델** — 1일 이내 사용분 차감 후 잔액 환불, 경과 후 결제 주기 종료까지 사용 보장 (`RQ-PAY-408·410·411`).

## 5. 성공 지표 (KPI)

> 모든 KPI는 측정 가능 / 검증 가능 형태로 정의한다 (PM 운영 원칙 §3 — 측정 가능 원칙).

### 5.1. 1차 (MVP — 시장 진입 확인)

| 분류 | 지표 | 목표 (출시 후 3개월) | 측정 출처 |
|---|---|---|---|
| **획득** | 누적 가입 회원 수 (개인 + 사업자) | 500명 | `RQ-ADMIN-201` |
| **획득** | 사업자 가입 심사 평균 처리 시간 | 24시간 이내 | `RQ-ADMIN-101~104` |
| **활성** | 운영 키 전환 자동 승인 비율 | 70% 이상 | `RQ-KEY-009` 로그 |
| **활성** | 체험 모드 → 회원가입 전환율 | 15% 이상 | `RQ-TRY-015` |
| **활성** | MAU(월간 활성 회원, 월 1회 이상 발송) | 200명 | 발송 로그 |
| **수익** | 누적 캐시 충전액 | 5,000만 원 | `RQ-PAY-001` |
| **수익** | 회원 1인당 평균 월 발송액 (ARPU) | 30,000원 | 결제 로그 |

### 5.2. 1차 (MVP — 차별화 검증)

| 분류 | 지표 | 목표 |
|---|---|---|
| **AI 통합** | CLI 호출 비중 (AI Agent 발송·조회 중 CLI 위임 비율) | 50% 이상 |
| **AI 통합** | MCP 조회·발송 도구 동등성 충족도 (가이드·발송·이력·발신번호·키 목록·잔액·사용·템플릿·브랜드 조회) | 100% — **단, 보안·재무 민감 액션(키 발급/폐기, 발신번호 등록/삭제, 결제·환불, 스코프 변경)은 의도적 MCP 비노출 (RQ-MCP-014). CLI 명시 명령으로만 수행** |
| **보안** | 키별 스코프 적용 비율 (`scope:key` 기본 활성) | 95% 이상 |
| **운영** | 운영자 수동 검토 큐 평균 처리 시간 | 4시간 이내 (`RQ-ADMIN-603`) |

### 5.3. 운영 품질 / SLA

| 분류 | 지표 | 목표 |
|---|---|---|
| **가용성** | 발송 API 가용성 | 99.5% 이상 |
| **정확성** | 발송 테이블 적재 성공률 | 99.9% 이상 |
| **응답성** | 발송 API 평균 응답 시간 | 500ms 이하 |
| **보안** | 이상 패턴 자동 차단 false-negative | 5% 이하 (`RQ-SEC-006`) |
| **재무** | 자동충전 결제 실패율 | 1% 이하 (`RQ-ADMIN-509`) |
| **CS** | 1:1 문의 평균 응답 시간 | 24시간 이내 (`RQ-ADMIN-702`) |

### 5.4. 비대상 (1차 KPI 아님)

- AI 토큰 비용 — 본 서비스가 측정·과금하지 않음 (`§6 인증·과금 원칙`).
- 글로벌 발송량 — `RQ-ARCH-006` 확장 설계만, MVP 측정 X.

## 6. MVP 범위 / 비대상 범위

### 6.1. MVP 포함

- `RQ-AUTH` 개인·일반 사업자 가입 (특수부가·사설 연동 제외)
- `RQ-TRY` 체험 모드 전체
- `RQ-MEMBER` 회사 마스터 / 하위 계정 (A/B안 중 **1안 결정 필요** — `§9.2 미확정 결정`)
- `RQ-ADMIN` 심사·회원 통제·발송 통제·라우팅·재무·테스트·CS·시스템 설정
- `RQ-CALLBACK` 개인·임직원·법인 대표번호 등록 (KISA 사전 등록 자동 연계)
- `RQ-KEY` 키 발급·스코프·자동 승인
- `RQ-SDK` Python (1차)
- `RQ-MCP` 가이드·스니펫·발송·조회 (보안 민감 비노출)
- `RQ-CLI` 전체 (`wsc` 명령 — 설치·인증·발송·조회·도움말·자동 업데이트)
- `RQ-SEND` 정합성 검증 → 인코딩 변환 → 발송 테이블 적재 (외부 발송 시스템 연계 인터페이스 정의)
- `RQ-TPL` 카카오 알림톡 / RCS 브랜드 (중계사 연동만, 직접 대행 X)
- `RQ-PAY` 선불 충전 + 잔액 부족 분기 + 자동충전 + 후불 + 구독 (§B·§C·§E 1차 도입 확정 — 2026-04-30)
- `RQ-TEST` 채널별 테스트 한도 자동 부여
- `RQ-SEC` 스팸 필터·이상 패턴·한도 차단·자동 해제 신청
- `RQ-OPS` FAQ·챗봇·1:1 문의·공지
- `RQ-ARCH` 3채널 통합·UTF-8 인입·EUC-KR 내부

### 6.2. MVP 비대상

- 특수부가 사업자 / 사설 연동 (`RQ-AUTH-201~202`, `RQ-CALLBACK-007·203`).
- 카카오 직접 대행사 등록 (`§14.D` — B안 확정).
- 글로벌 발송 (`RQ-ARCH-006` — 확장만).
- 자사 발송 시스템 자체 구축 (외부 시스템 책임 — `§7.4`).

## 7. 비즈니스 모델 / 과금

| 항목 | 정책 |
|---|---|
| **기본 모델** | 선불 캐시 충전 → 채널별 단가 차감 |
| **무료 진입점** | SDK 패키지·CLI 바이너리·정적 가이드/스니펫 (`docs`/`snippet`) — 무인증·무과금 |
| **자원 호출** | 발송·조회·잔액 — API Key 인증 필수, 발송 시 캐시 차감 |
| **AI 토큰 비용** | 사용자 본인 AI Agent 부담 — 본 서비스가 측정·과금하지 않음 |
| **웹 무료 발송** | 미제공 (`RQ-SEC-001`, `RQ-ADMIN-308`) — "무료 50건" 식 어뷰징 진입점 차단 |
| **SDK/MCP/CLI 테스트 발송** | **무과금** — 테스트 키 발급 시 채널별 테스트 발송 한도가 자동 부여(`RQ-TEST-005~009`)되며, 운영 키 전환 전까지 한도 내 무과금 (실사용 캐시 차감 X). 한도 소진 시 차단 + 리셋 시점 안내 (`RQ-TEST-007·008`). 한도 정책은 슈퍼 어드민 관리(`RQ-ADMIN-612`) |
| **무통장입금** | 미지원 (`RQ-PAY-007`) |
| **옵션 상품 (도입 시)** | 자동충전(§B)·후불(§E)·구독(§C) — 1차 도입 여부는 결정 필요 |

## 8. 마일스톤 / 로드맵 (개략)

> 정확한 일정·인력 배분은 마스터 플랜 + WBS에서 확정한다. 본 PRD는 단계 의도만 적시.

| 단계 | 핵심 산출물 | 게이트 |
|---|---|---|
| **M1 — 회원·발신번호·키 기반** | `RQ-AUTH`·`RQ-MEMBER`·`RQ-CALLBACK`·`RQ-KEY` 동작 | 사업자 심사 24h, 자동 승인 70% |
| **M2 — 발송 파이프라인** | `RQ-SEND` 적재 + 인코딩 변환 + 외부 시스템 인터페이스 합의 | 발송 적재 성공률 99.9% |
| **M3 — SDK·CLI·MCP** | `RQ-SDK`·`RQ-CLI`·`RQ-MCP` (조회·발송 동등) | MCP↔CLI 1:1 동등성 100% |
| **M4 — 결제·잔액·체험 모드** | `RQ-PAY` 선불 + 잔액 분기 + 자동충전 + 후불 + 구독 + `RQ-TRY` | 체험→가입 전환율 15%, 자동충전 결제 실패율 ≤ 1% |
| **M5 — 운영자 콘솔·보안** | `RQ-ADMIN`·`RQ-SEC`·`RQ-OPS` | 1:1 문의 24h 처리 |

## 9. 가정 / 미확정 결정

### 9.1. 가정

- 외부 발송 시스템(통신사·중계사 연동·발송 에이전트)은 별도 팀이 구축한다. 본 서비스는 발송 테이블 인터페이스로 연계.
- KISA 사전 등록 API는 회원의 발신번호 등록 시 자동 연계 가능한 형태로 제공된다.
- 카카오·RCS 중계사(LG CNS·KT·인포뱅크)와의 계약·매핑은 운영자가 사전 확보.

### 9.2. 1차 출시 전 결정 (PM 안건)

| 안건 | 결정 | 영향 | 참조 |
|---|---|---|---|
| **§A. 회사 마스터 권한 구조** | ✅ **A안 권한 부여형 확정 (2026-04-30)** | 권한 이관·세분 권한·확장성 우위. B안 전용 RQ-MEMBER-301~303 1차 범위 외 처리 | `SRS §14.A` |
