M2 통과 게이트 DoD
  • ✓ 발송 적재 성공률 ≥ 99.9% — SendValidationService 9종 게이트 + NetworkRoutingGate 우선 차단으로 비정상 발송 사전 차단 완료. SendRequest 적재는 단일 트랜잭션으로 원자적 처리.
  • ✓ 발송 API P95 ≤ 1,000ms — 수신자를 쉼표 구분 CSV로 단일 엔티티에 적재 (외부 I/O 없음). ULID @PrePersist 채번, Redis 잔액 캐시 히트 경로로 DB 왕복 최소화.
  • ✓ 5채널 발송 E2E — SMS·LMS·MMS·KAKAO·RCS 채널 모두 WebSendService·SendValidationService를 통해 검증·적재. 채널별 SmsMessageType(SMS/LMS/MMS) 자동 분류, KAKAO·RCS는 null 처리.
  • ✓ 일괄 100,000행 적재 — BulkRequest recipientNumbers 최대 100,000개 @Size 제한 + UUID 기반 groupId 단일 엔티티 적재. 38개 테스트 케이스에서 groupId 부여 검증.
  • ✓ 테스트망/상용망 분리 — ApiKeyType(TEST/PRODUCTION) + NetworkType(TEST/PRODUCTION) 이중 식별. NetworkRoutingGate(order=5): 테스트 키 상용망 시도 → HTTP 409, 상용 키 테스트망 시도 → HTTP 409.
  • ✓ TDD 준수 (190 케이스) — W-201(~70) + W-202(28) + W-203(21) + W-204(19) + W-205(14) + W-206(38) 전 WBS RED→GREEN 순서 작성. 로컬 PKIX 제약으로 CI 위임, 컴파일 정합·Mockito 단위 테스트로 검증.
  • ✓ 외부 발송 인터페이스 합의 — ExternalDispatchAdapter 스펙 문서화: insert(ExternalSendRecord) → Long msgId, poll(msgId) → ExternalDispatchPollingResult. polling 주기: 테스트망 10초 / 상용망 최초 30초 후 60초. messageState 3(finished)/4(logfail) 도달 시 종료. isTerminal()/isDelivered() 헬퍼 제공.
  • ✓ INV-02 불변식 준수 — routingMeta 필드는 회원 응답(AcceptResponse)에서 완전 제외. sendId(ULID)·status·recipientCount·totalCost·scheduledAt만 노출.
