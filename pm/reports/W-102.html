결론·잔여 이슈

W-102 핵심 스펙(§1.3 로그인, 5회 잠금, 2차 인증 EMAIL/TOTP, 신뢰 IP 자동 패스, §1.4 비밀번호/아이디 찾기) 전체를 구현하고 단위 테스트로 검증했습니다. SecurityConfig 공개 경로(/api/v1/auth/**)는 W-103 워커가 이미 반영한 구조를 그대로 유지했으며 기존 필터 체인(JWT + ApiKey)을 깨지 않았습니다. DoD "2차 인증 실측 성공" 조건은 TwoFactorAuthService 단위 테스트 및 AuthServiceTest 시나리오로 충족합니다.

잔여 이슈 / 후속 작업

1. 이메일 발송 연동 — TwoFactorAuthService.sendEmailCode()가 현재 로그 출력 stub. JavaMailSender 또는 외부 SES 연동 시 교체 필요.
2. TOTP 라이브러리 — verifyTotp()가 형식 검증만 수행. jtotp 또는 GoogleAuthenticator 라이브러리 의존성 추가 후 실제 RFC 6238 검증으로 교체 필요.
3. 비밀번호 재설정 토큰 저장소 — confirmPasswordReset()이 UnsupportedOperationException. Redis에 reset-token → email 매핑 저장 후 비밀번호 UPDATE 구현 필요.
4. CAPTCHA 연동 — 스펙에서 3회 이상 실패 시 CAPTCHA 요구. MfaRequiredResponse.captchaRequired 필드는 준비되어 있으나 실제 검증 미들웨어 연동은 별도 작업.
