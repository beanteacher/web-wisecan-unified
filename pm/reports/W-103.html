결론·잔여 이슈
W-103의 모든 DoD 항목이 충족되었습니다. 회사 마스터 권한 부여·이관·회수 4가지 시나리오(GRANT / TRANSFER / AUTO_TRANSFER / REVOKE)가 단일 트랜잭션으로 구현되었고, COMPANY_ROLE_LOG에 모든 사건이 기록됩니다. 하위 계정은 MEMBER와 분리된 독립 테이블(COMPANY_SUB_ACCOUNT)로 관리되며, 초대 링크(SHA-256 tokenHash, 7일 유효) 수락 시 자동 생성됩니다. SecurityConfig는 공개 경로 1건만 최소 추가하여 W-102/104/105와의 충돌을 방지했습니다. 서비스 단위 테스트 13종이 Mockito로 작성되었습니다.
잔여 이슈 및 후속 작업
  • 로컬 SSL 인터셉션 환경으로 실제 빌드·테스트 실행은 CI에 위임 — GitHub Actions에서 검증 필요
  • 하위 계정 삭제 시 해당 계정 명의 발신번호 연쇄 삭제(§3.3) — W-104(발신번호)와 이벤트 연동 필요 (ApplicationEventPublisher 또는 서비스 간 호출)
  • 마스터 후보 선출 쿼리가 현재 findAll() 후 스트림 필터 방식 — 운영 단계에서 DB 인덱스 활용 JPQL 쿼리로 교체 권장
  • MySQL 8+ 부분 UNIQUE 인덱스(virtual generated column)는 Flyway/Liquibase 마이그레이션 스크립트로 별도 적용 필요
  • 초대 이메일 발송 로직 미구현 — 알림 도메인(W-106 이후) 연동 시 추가
