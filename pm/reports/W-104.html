등록 케이스 개요 (02_FEATURE_SPEC §4.1 ~ §4.4)
§4.1 — 케이스 1
SELF_MOBILE
본인 인증 휴대폰 번호와 입력 번호가 일치하는 경우 즉시 등록. 번호 불일치 시 400 오류.
결과 상태: REGISTERED (즉시)
§4.2 — 케이스 2
SELF_LANDLINE
비-휴대폰(유선) 번호 즉시 등록. 통신사 명의 확인은 외부 연동(스텁)으로 처리.
결과 상태: REGISTERED (즉시)
§4.3 — 케이스 3
EMPLOYEE
임직원 번호 심사형 등록. 재직 증명 서류 업로드 후 운영자 심사 큐에 배치.
결과 상태: SUBMITTED (심사 대기)
§4.3 — 케이스 4
CORP_REP
법인 대표번호 심사형 등록. 통신 서비스 이용 확인서 등 법인 서류 제출 후 심사 큐 배치.
결과 상태: SUBMITTED (심사 대기)

상태 머신

SUBMITTED → UNDER_REVIEW → REGISTERED 또는 REJECTED
REGISTERED → DELETED 케이스 1·2는 SUBMITTED 건너뜀 (즉시 REGISTERED)
DoD 및 충족 여부
완료 기준 상태 비고
4 케이스 등록 성공 (SELF_MOBILE / SELF_LANDLINE / EMPLOYEE / CORP_REP) 완료 단일 진입점 POST /api/v1/callbacks, registerType 분기
즉시 등록(§4.1·4.2) 결과 상태 REGISTERED 확인 완료 registerImmediately() 도메인 메서드 + CallbackLog 기록
심사형(§4.3) 결과 상태 SUBMITTED 확인 완료 EMPLOYEE / CORP_REP 모두 SUBMITTED 저장
발신번호 삭제 (§4.4) — DELETED 상태 전환 완료 DELETE /api/v1/callbacks/{id}, 본인 소유 검증 포함

결론 및 잔여 이슈

W-104 핵심 범위인 발신번호 등록 4 케이스(SELF_MOBILE·SELF_LANDLINE·EMPLOYEE·CORP_REP)의 BE 도메인·서비스·컨트롤러·테스트가 완료되었습니다. 단일 진입점(POST /api/v1/callbacks)에서 registerType으로 분기하여 즉시 등록(REGISTERED)과 심사 큐(SUBMITTED) 흐름을 분리하였으며, 활성 등록 1개 강제, 연쇄 삭제, append-only 감사 로그가 모두 구현되었습니다. W-106(운영자 콘솔 검토 큐)의 심사 진입 데이터 소스로 사용 가능한 상태입니다.

잔여 이슈 (다음 스프린트 또는 별도 태스크)
  • FE 발신번호 등록 화면 — 4 케이스 분기 UI, 서류 업로드 폼 (BE API 완료 후 연동 가능)
  • CallbackDocument 서류 파일 업로드 — 파일 스토리지(S3·MinIO 등) 연동 별도 태스크 필요
  • 통신사 명의 확인 외부 연동(SELF_LANDLINE) — 외부 계약 후 구현, 현재 스텁 처리
  • 심사 전환(UNDER_REVIEW → REGISTERED/REJECTED) 운영자 API — W-106 태스크에서 구현
  • 로컬 빌드 실행 검증 — SSL 인터셉션 해소 후 CI 파이프라인에서 확인
  • SELF_MOBILE 본인 인증 번호 취득 경로 — PASS/NICE 연동(W-101 잔여) 완료 후 연계
