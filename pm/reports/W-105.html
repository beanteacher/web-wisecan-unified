결론 및 잔여 이슈

W-105 DoD 전 항목(12종 스코프 카탈로그·권장 프리셋 4종·발급·스코프 수정·폐기·재발급 흐름)이 BE+FE 레이어 모두에서 구현·검증 완료되었습니다. 기존 ApiKeyAuthFilter의 스코프 하드코딩이 제거되어 DB에 저장된 실제 스코프 기반으로 동작합니다. W-102 완료 이후 JWT 인증과의 통합 E2E 테스트가 필요합니다.

잔여 이슈 / 후속 과제
  • 스코프 기반 API 접근 제어 미들웨어 구현 필요 — 현재 ApiKeyAuthFilter는 스코프를 SecurityContext에 주입만 하고, 각 엔드포인트에서 스코프를 실제로 검증하는 로직(예: @PreAuthorize 또는 별도 인터셉터)은 W-107 이후 구현 예정
  • SCOPE_NOT_GRANTED 403 응답 형식 표준화 — 02 §5.3 예외("미허용 스코프 호출 → SCOPE_NOT_GRANTED 403") 처리를 ErrorCode enum에 추가하고 GlobalExceptionHandler에 연동 필요
  • FE 스코프 선택 UI 미구현 — IssueApiKeyDialog에서 스코프 다중 선택 체크박스는 현재 기본값(keyName만 입력) 사용 중. W-106 이후 키 발급 UI 개선 스프린트에서 추가 예정
  • 로컬 빌드 PKIX 오류로 gradlew test 직접 실행 불가 — CI 환경에서 검증 필요 (env_local_ssl_interception 정책 참조)
