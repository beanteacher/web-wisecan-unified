결론·잔여 이슈

W-106 핵심 스펙(§12.1 사업자 전환 심사, §12.2 발신번호 심사, §12.6 API Key 검토 큐) 전체를 AdminReviewService + AdminReviewController로 구현하고 단위 테스트 12개로 검증했습니다. W-103·104·105 산출물(BusinessApplication, Callback, ApiKey 도메인)을 재사용하여 Company 생성 + Member 역할 전이를 단일 트랜잭션으로 처리했으며, DoD "큐 처리 → 회원 알림 E2E" 중 큐 처리 로직은 완료되었습니다.

잔여 이슈 / 후속 작업

1. 회원 알림 연동 — 승인·반려 시 회원에게 이메일/앱 알림 발송이 스펙에 명시되어 있으나 현재 로그 출력만 수행. ApplicationEventPublisher + 알림 서비스 연동 필요.
2. AdminId 추출 — AdminReviewController.resolveAdminId()가 현재 0L 반환 stub. JWT UserDetails에서 memberId를 가져오는 로직 구현 필요.
3. ADMIN 역할 인가 — 현재 authenticated() 수준만 적용. @PreAuthorize("hasRole('ADMIN')") 또는 MemberRole.ADMIN 체크 추가 필요.
4. 통화 녹음 저장 — §12.1 스펙의 "통화 녹음 보존 5년" 항목은 미구현 (별도 스토리지 연동 작업).
