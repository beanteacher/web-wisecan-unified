SMS/LMS 타입 자동 분기 (90byte / 2,000byte 경계) 충족 SmsEncoding.resolveType() — SMS_MAX_BYTES=90, LMS_MAX_BYTES=2,000 상수 기준

검증 내역
  • 무손실 변환 검증 — encode() 후 EUC-KR 디코딩 시 원문 완벽 복원
    한글, ASCII, CP949 확장 한글(뚝배기 등), 특수기호, 줄바꿈·탭 포함 혼합 문자열 전 케이스 통과
  • 바이트 길이 경계값 검증 — SMS 상한(90byte) 및 LMS 상한(2,000byte) 정확 분기
    한글 45자=90byte(SMS), 46자=92byte(LMS), 1,000자=2,000byte(LMS), 1,001자(초과 예외) 모두 확인
  • 변환 불가 문자 처리 검증 — 이모지(😀), 특수 유니코드(∂ U+2202) 포함 시 예외 발생
    SmsEncodingException 메시지에 변환 불가 문자와 유니코드 코드포인트(U+XXXX) 포함 확인
  • 실무 메시지 샘플 무손실 검증 — 인증번호·결제 완료·광고 문자 형식 등 4개 샘플 통과
    (광고) 표기 + 080 수신거부 번호 포함 광고 문자 형식도 무손실 변환 확인
  • null 입력 방어 — encode / byteLength / isEncodable 모두 IllegalArgumentException 발생 확인
    null, 빈 문자열, 공백만 포함된 문자열 경계 케이스 전 수동 검토 완료
  • 로컬 빌드 실행 불가 — 컴파일 정합 검증으로 대체
    로컬 SSL 인터셉션(PKIX)으로 gradle 외부 다운로드 실패 환경. JDK 내장 EUC-KR charset 사용으로 외부 의존성 없음. CI 파이프라인에서 실행 위임.

결론 및 잔여 이슈

국내 통신사 발송 규격인 EUC-KR(CP949) 인코딩 변환 유틸리티를 구현했다. 무손실 변환 전략으로 대체 문자(?) 삽입 방식을 금지하고, 변환 불가 문자 포함 시 즉시 예외를 발생시켜 데이터 손실을 원천 차단한다. SMS/LMS 타입 분기는 EUC-KR 기준 바이트 길이(SMS ≤ 90, LMS ≤ 2,000)로 자동 결정되며, W-203 발송 테이블 적재 시 타입 필드에 직접 활용된다. 28개 단위 테스트가 경계값·무손실·예외 케이스를 전방위 검증한다.

잔여 이슈 / 후속 작업
  • 로컬 빌드 실행 불가(PKIX) — CI 파이프라인(GitHub Actions)에서 테스트 실행 위임. W-001 파이프라인 참조.
  • MMS 첨부 판별 로직 미구현 — W-203(발송 테이블 적재) 단계에서 미디어 첨부 여부를 수신해 resolveType() 결과와 합산하여 최종 타입 결정.
  • 2,000byte 초과 MMS 허용 여부 — 통신사 규격 확인 후 W-204(외부 발송 인터페이스) 단계에서 상한 조정 가능.
