결론 및 잔여 이슈

W-203 발송 테이블 적재 + 메타·라우팅 구현을 완료했다. ULID 단일 식별자(외부 라이브러리 없는 순수 Java 구현), SendRequest 엔티티(채널·발신번호·수신자·본문·예약시각·광고·카카오 메타·라우팅 예약 필드 전 항목), W-201 검증·W-202 인코딩 연계 흐름, Repository·Service·Controller·DTO 전 레이어를 TDD 기반으로 구현했다. 로컬 빌드 PKIX 이슈 대응으로 외부 ULID 라이브러리 대신 표준 Java API만 사용해 의존성을 추가하지 않았다.

잔여 이슈 (W-204에서 처리 예정)
  • 외부 발송 시스템 INSERT(message_state=0) 미구현 — 현재 status=PENDING 반환. W-204에서 외부 DB 스키마 연동 후 markQueued() 호출 추가
  • 잔액 차감(§7 CHARGE_BALANCE_LEDGER) 미연계 — W-204에서 외부 INSERT 직전 차감, 실패 시 REVERT 보상 트랜잭션 구현
  • routingMeta 산출 로직 미구현 — W-204에서 routing:{memberId} Redis 캐시 기반 중계사 매핑 값 산출 후 updateRoutingMeta() 업데이트
  • 단가(unitCost) 임시 헤더 수신 (X-Unit-Cost) — W-204에서 채널별 단가 테이블 조회로 교체
  • 적재 P95 ≤ 1s 외부 I/O 포함 재측정 — W-204 완료 후 부하 테스트 필요
