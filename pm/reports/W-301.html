결론 및 잔여 이슈

W-301 Python SDK 구현이 완료되었습니다. sdk/python/ 신규 디렉토리에 패키지 소스(5모듈), pytest 단위 테스트(62개 전체 통과), pyproject.toml(PyPI 배포 메타), README.md를 포함한 총 11개 파일이 생성되었습니다. 백엔드 SendChannel 5종(SMS/LMS/MMS/KAKAO/RCS)과 SendRequestStatus enum이 1:1 대응하며, WebSendController /console/send 계열 API 계약을 준수합니다. API Key 접두사(wsc_live_ / wsc_test_)로 운영·테스트 환경이 자동 분기되고, HTTP 오류 상태 코드별 예외가 정확히 매핑됩니다.

잔여 이슈 / 후속 작업
  • 실제 PyPI publish는 운영 배포 시점(M5 릴리즈 게이트)에서 수행 — 현재는 로컬 editable 설치만 검증
  • 비동기(asyncio) 클라이언트 미구현 — httpx.AsyncClient 기반 AsyncWiseCan 추가는 1차 외 범위(02 §7.1 "Python 1차")
  • MMS 첨부 파일(이미지) 업로드 API는 백엔드 엔드포인트 미확정 — 엔드포인트 확정 후 send_mms에 media_url 파라미터 추가 필요
  • 로컬 SSL 인터셉션 환경 — 외부 httpx 설치는 이미 완료된 환경 의존, CI 파이프라인에서 재검증 권장
