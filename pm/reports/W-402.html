자동충전 트리거 흐름
1
잔액 임계 도달 감지

발송 차감(deductForSend) 후 또는 주기 배치에서 triggerIfNeeded(memberId, currentBalance) 호출

2
설정 유효성 검증

AutoChargeConfig 조회 → 활성(enabledYn=Y) + 미만료(expiresAt) 확인. 만료 시 자동 비활성화 후 스킵

3
결제수단·빌링키 확인

PaymentMethod 조회 → activeYn=Y + pgBillingKey 존재 확인. 없으면 경고 로그 후 스킵

4
PG 정기결제 요청

PgPaymentPort.requestPayment(pgBillingKey, chargeAmount, idempotencyKey). 멱등성 키 = "AUTO-{memberId}-{UUID}"

5
결과 처리

성공: CHARGE(AUTO) + CHARGE_BALANCE 저장, BalanceCacheEvictEvent 발행
실패: CHARGE(FAILED) 저장, AutoChargeFailedEvent 발행 → AFTER_COMMIT 알림

결론 및 잔여 이슈

W-401 billing 도메인(PgPaymentPort·ChargeService·ChargeBalance)을 재사용해 자동충전 전용 파일만 신규 추가했습니다. 공유 파일(ChargeService 등) 수정은 0건으로 W-403·W-404와의 충돌 위험 없음. 임계 도달 트리거·30일 제한·실패 알림 이벤트 모두 TDD(RED→GREEN) 방식으로 구현·검증 완료. DoD 전 항목 충족.

잔여 이슈 / 후속 작업
  • AutoChargeFailedListener — 실 알림 연동 (SMS/이메일/알림톡) 구현 필요 (알림 모듈 W-5xx 연동 시점에 완성).
  • triggerIfNeeded 호출 시점 — 현재 호출처는 명시적 외부 호출 전제. 배치 스케줄러(@Scheduled) 또는 발송 차감 직후 호출 연결은 W-405(잔액 부족 분기)에서 확정.
  • AutoChargeController X-Member-Id 헤더 — JWT 인증 필터 완성 시 SecurityContext 기반으로 교체 필요.
  • CI 빌드 정합 검증 — 로컬 SSL 인터셉션으로 Gradle 빌드 불가. GitHub Actions CI에서 최종 컴파일·테스트 실행 필요.
