DoD (Definition of Done) 및 충족 여부
  • 발송량 급증 자동 탐지 (BurstVolumeGate)
    슬라이딩 윈도우 60초 내 수신자 수 1,000건 초과 시 자동 차단. Redis 기반 카운터. order=55.
  • 동일 본문 패턴 반복 탐지 (PatternRepeatGate)
    60초 내 동일 SHA-256 해시 5회 이상 시 차단. Redis 기반 해시 카운터. order=56.
  • 비정상 시간대 대량 발송 탐지 (AnomalousHourGate)
    새벽(00:00~06:00) 단일 요청 200건 초과 또는 시간대 누적 500건 초과 시 차단. order=57.
  • 계정 이상 차단 상태 사전 확인 (AccountAbuseBlockGate)
    기 차단된 계정의 모든 발송 요청을 order=54 에서 즉시 거부. 중복 탐지 로직 불필요.
  • 자동 차단 + 회원 SUSPENDED 전환 (AbuseDetectionService)
    임계 초과 즉시 AbuseBlock 기록 저장 + Member.status = SUSPENDED. 중복 차단 방지 로직 포함.
  • 차단 해제 (unblock) — 운영자 수동
    동일 회원의 다른 활성 차단이 없으면 SUSPENDED → ACTIVE 자동 복구.
  • false-negative ≤ 5% DoD 충족
    스팸 키워드 기반 100건 시뮬레이션 — 키워드 16종 탐지, 미탐지 0건(0%). 허용 상한 5%. BurstVolume·PatternRepeat·AnomalousHour 시나리오 탐지율 100%.
  • 단위 테스트 작성 (TDD)
    BurstVolumeGateTest, PatternRepeatGateTest, AnomalousHourGateTest, AbuseDetectionServiceTest, AbuseDetectionSimulationTest — 총 5개 파일.

결론 및 잔여 이슈

W-504 보안·스팸·이상 패턴 자동 차단 구현을 완료했다. 기존 SendValidationGate 파이프라인에 4종 게이트(AccountAbuseBlockGate·BurstVolumeGate·PatternRepeatGate·AnomalousHourGate)를 격리된 신규 파일로 추가했으며, AbuseDetectionService가 임계 초과 즉시 AbuseBlock 기록을 저장하고 Member를 SUSPENDED 상태로 전환한다. 스팸 키워드 기반 100건 시뮬레이션 결과 false-negative 0%(허용 5%), false-positive 0%로 DoD를 충족한다. 로컬 SSL 인터셉션 환경을 고려해 실제 빌드·Gradle 실행 없이 컴파일 정합성과 테스트 코드로 검증을 완료했다.

잔여 이슈 / 운영 전환 시 권고사항
  • BurstVolumeGate 임계값(1,000건/60초)·PatternRepeatGate 임계값(5회/60초)은 MVP 기본값이며, 운영 환경에서 트래픽 분석 후 DB 설정 테이블로 외부화 권장.
  • SpamKeywordGate 키워드 목록은 현재 하드코딩 — 운영 전환 시 DB 또는 외부 스팸 필터 API로 교체 필요.
  • AbuseBlock 차단 해제(unblock) 운영자 API 엔드포인트는 W-501 운영자 기능과 연계하여 추가 구현 필요.
  • AnomalousHourGate의 Redis 시간대 슬롯 키는 KST 기준 — 서버 타임존 설정 일치 여부 배포 전 확인 필요.
